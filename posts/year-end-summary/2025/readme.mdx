当时其实没什么"我要参与开源""我要成为 Vue member"这种宏大叙事，更多就是单纯的：

- 这个地方我刚好看懂了
- 这个问题我好像也能修
- 那不如试试？

后来慢慢就变成了一种惯性：看 issue、翻源码、顺手修一下、提 PR。

回头一看才发现，已经在 Vue Core 里留下了一串记录：

- [commits](https://github.com/vuejs/core/commits/main/?author=linzhe141)
- [pull requests](https://github.com/vuejs/core/pulls/linzhe141)

嘴上说是"水 PR"，但实际上每个 PR 都逃不开这些过程：

- 读源码，理解上下文
- 搞清楚设计意图，而不是只修表象
- 被 review 指出问题，然后改
- 有些地方想当然了，后来才意识到「哦，原来这里不能这么动」

我还记得第一个 PR merge 之后，脑子里冒出来一个念头：

> 我是不是也有机会，成为 Vue members 里的一员？

那会儿其实不太敢当真，更像是一种自我激励。

结果到了今年，这件事真的发生了——我加入了 **Vue Organization**。

那一刻是真的激动。不只是那种表面的开心，而是一种被认可的感觉：

> 我这段时间做的事情，是有意义的。

## 做了一个有人用的 npm 包

除了 Vue Core，今年我还做了一件对自己来说挺重要的事：**做了一个真的有人在用的 npm 包**。

就是这个：[vue-mdr](https://github.com/linzhe141/vue-markdown-renderer)

让我觉得这件事"值了"的，不是 npm 上多了一个包，而是：

- 有人 star
- 有人提 issue
- 有人真的在项目里用，而不是只跑 demo

当你意识到**别人要为你的 API 设计负责**的时候，心态会明显不一样。你开始考虑兼容性、边界情况、使用体验，而不只是"能跑就行"。

## 其他碎片

这一年里：

- 能看的 F1 基本都看了，可惜阿布扎比没能看到维斯塔潘拿到 WDC 🏆
- 学习了 Electron，写了个 chat 应用，折腾了 LLM、SSE、ReadableStream

## 写在最后

如果用一句话概括 2025：

> **我从"主要使用技术的人"，变成了"开始参与技术本身的人"。**

这并不意味着我有多厉害，只是说明了一件事：

> 我开始愿意为一个系统负责一小块，而不只是用它。

走得不快，但方向是对的。

至少现在回头看，我不会觉得这一年是空的。
